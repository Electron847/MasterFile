CSC 402/Data Structures I
Timing implementations of union/find

Setting up

In the src directory in Eclipse, create a package called timeuf. Inside that package, create a class called TimeUF. This creates a shell program with a mostly empty main method. All of your code will go in the main method.

In the main method, declare a timer, which is a variable of type Stopwatch, and a count, which is a variable of type int. Ther are for UF classes in the algs14 package:

  • QuickFindUF
  • QuickUnionUF
  • WeightedUF
  • CompressionUF
For each one, write a block of code that:
  1. Directs StdIn to read from the file "data/uf16k.txt"
  2. Reads an integer from StdIn into the count
  3. Initializes the timer to a new Stopwatch object
  4. Declares and initalizes a new QuickUnionUF object, passing it the count as the size
  5. While StdIn is not empty (that is, while there is input to read):
    • Reads an integer from to StdIn into a variable p
    • Reads an integer from to StdIn into a variable q
    • Unions p and q using the union method of the QuickUnionUF object
  6. Prints to StdOut how long this took by using the elapsedTimer method on the timer variable
You will also have to download the data file uf16k.txt and drag it into the data directory in Eclipse.

Use the sample program TimeLists.java for an example of how to do most of what is written above. Its main method contains three blocks of code, each one timing a different list implementation from the classes defined in the algs13 package. In Eclipse, create a package called csc402 and drag TimeLists.java into it. From there, you should be able to run it. It uses a data file called 32KInts.txt that you download and drag into the data directory.

To give you a more concrete idea of what I expect, here is the output from my solution. Note that your times may differ.
